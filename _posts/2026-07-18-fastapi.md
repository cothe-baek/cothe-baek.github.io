FastAPI 애플리케이션의 실행 구조와 기본 요청 처리 방법 정리

## 1. FastAPI

FastAPI는 Python으로 API 서버를 개발하기 위한 웹 프레임워크

주요 기능

* URL과 Python 함수 연결
* 요청 데이터 변환
* 입력값 검증
* JSON 응답 생성
* 비동기 처리 지원
* API 문서 자동 생성

기본 코드

```python
from fastapi import FastAPI

app = FastAPI()


@app.get("/")
def read_root():
    return {"message": "Hello World"}
```

### `app = FastAPI()`

FastAPI 애플리케이션 객체 생성

URL, HTTP 메서드, 실행할 함수 등의 정보가 `app`에 등록됨

### `@app.get("/")`

`GET /` 요청이 들어오면 아래 함수를 실행하도록 연결하는 데코레이터

```text
GET /
→ read_root()
```

### 반환값

```python
return {"message": "Hello World"}
```

Python 딕셔너리를 FastAPI가 JSON 응답으로 변환

```json
{
  "message": "Hello World"
}
```

## 2. FastAPI와 Uvicorn

FastAPI 코드 → 요청을 어떻게 처리할지만 정의된 상태

실제로 네트워크 요청을 받으려면 **서버 프로그램 (Uvicorn)**이 필요함

### FastAPI

요청 처리 방법 정의

* 어떤 URL을 사용할지
* 어떤 HTTP 메서드를 사용할지
* 어떤 함수를 실행할지
* 어떤 데이터를 반환할지

### Uvicorn

FastAPI 애플리케이션을 실행하는 ASGI 서버

* 포트 열기
* HTTP 요청 수신
* FastAPI에 요청 전달
* FastAPI 응답을 클라이언트에 반환

실행 명령어

```bash
uvicorn main:app --reload
```

| 구성         | 의미                |
| ---------- | ----------------- |
| `uvicorn`  | Uvicorn 서버 실행     |
| `main`     | `main.py` 파일      |
| `app`      | 파일 내부의 FastAPI 객체 |
| `--reload` | 코드 변경 시 서버 자동 재시작 |

파일명이 `server.py`, FastAPI 객체 이름이 `api`라면

```bash
uvicorn server:api --reload
```

`--reload`는 개발 환경에서 사용

운영 환경에서는 일반적으로 사용하지 않음

## 3. 요청 처리 흐름

클라이언트 요청

```text
http://127.0.0.1:8000/items/10?q=apple
```

FastAPI 코드

```python
@app.get("/items/{item_id}")
def read_item(item_id: int, q: str | None = None):
    return {
        "item_id": item_id,
        "q": q,
    }
```

전체 흐름

| 주체        | 처리 내용                           | 예시                          |
| --------- | ------------------------------- | ---------------------------------- |
| 클라이언트     | 서버에 HTTP 요청 전송                  | `GET /items/10?q=apple`            |
| Uvicorn   | 네트워크로 요청 수신, FastAPI에 전달        | `uvicorn main:app --reload`        |
| FastAPI   | URL과 HTTP 메서드에 맞는 함수 탐색         | `@app.get("/items/{item_id}")`     |
| Python 함수 | 요청 데이터로 함수 실행 후 결과 반환           | `read_item(item_id=10, q="apple")` |
| FastAPI   | Python 객체를 JSON 기반 HTTP 응답으로 변환 | `{"item_id": 10, "q": "apple"}`    |
| Uvicorn   | 완성된 HTTP 응답을 클라이언트에 전송          | `HTTP/1.1 200 OK`                  |
| 클라이언트     | 응답을 받아 화면 표시 또는 데이터 처리          | JSON 응답 사용                         |



함수 실행

```python
read_item(item_id=10, q="apple")
```

응답

```json
{
  "item_id": 10,
  "q": "apple"
}
```

## 4. WSGI와 ASGI

WSGI와 ASGI는 웹 서버와 Python 웹 애플리케이션이 통신하기 위한 인터페이스 규격

| 구분       | WSGI                         | ASGI                                  |
| -------- | ---------------------------- | ------------------------------------- |
| 전체 이름    | Web Server Gateway Interface | Asynchronous Server Gateway Interface |
| 처리 방식    | 동기 중심                        | 동기와 비동기 지원                            |
| 대표 서버    | Gunicorn, uWSGI              | Uvicorn, Hypercorn                    |
| 대표 프레임워크 | Flask                        | FastAPI, Starlette                    |
| 지원 범위    | 일반 HTTP 요청                   | HTTP, WebSocket, 스트리밍 등               |

Django는 WSGI와 ASGI 모두 지원

FastAPI와 Uvicorn의 관계

```text
클라이언트
→ Uvicorn
→ ASGI 규격
→ FastAPI
```

* FastAPI: ASGI 애플리케이션
* Uvicorn: ASGI 서버
* ASGI: 두 프로그램이 통신하기 위한 규격

## 5. 동기와 비동기

### 동기 처리

현재 작업이 끝난 뒤 다음 작업 처리

```python
def get_user():
    result = database_query()
    return result
```

데이터베이스 응답이 올 때까지 현재 작업 대기

### 비동기 처리

현재 작업이 응답을 기다리는 동안 다른 작업 처리 가능

```python
async def get_user():
    result = await database_query()
    return result
```

`await`는 결과가 완료될 때까지 기다리면서 이벤트 루프에 제어권을 반환

```text
작업 A가 입출력 대기
→ 작업 B 처리
→ 작업 A 결과 도착
→ 작업 A 처리 재개
```

비동기가 효과적인 작업

* 데이터베이스 요청
* 외부 API 요청
* 파일 입출력
* 네트워크 통신

비동기 처리를 사용하려면 해당 라이브러리도 비동기를 지원해야 함

```python
result = await async_database.find_all()
```

비동기는 한 요청의 처리 시간을 무조건 줄이는 방식이 아님

입출력 대기 시간 동안 다른 요청을 처리해 서버의 동시 처리 성능을 높이는 방식

## 6. `def`와 `async def`

FastAPI에서는 일반 함수와 비동기 함수 모두 사용 가능

### 동기 함수

```python
@app.get("/users")
def get_users():
    return sync_database.find_all()
```

동기 방식으로 동작하는 라이브러리 사용 시 적합

FastAPI는 `def`로 작성된 경로 함수를 별도의 스레드 풀에서 실행

### 비동기 함수

```python
@app.get("/users")
async def get_users():
    return await async_database.find_all()
```

비동기 방식으로 동작하는 라이브러리 사용 시 적합

구분 기준

```text
호출할 라이브러리가 동기 방식
→ def

호출할 라이브러리가 비동기 방식이고 await 사용 가능
→ async def
```

### 비동기 함수 내부의 블로킹 코드

`async def` 내부에서 동기 대기 함수를 사용한 경우

```python
import time


@app.get("/bad")
async def bad_endpoint():
    time.sleep(3)  # 이벤트 루프를 막는 동기 대기
    return {"message": "done"}
```

`time.sleep()`이 실행되는 동안 이벤트 루프가 막혀 다른 비동기 작업 처리 불가

비동기 대기 함수 사용

```python
import asyncio


@app.get("/good")
async def good_endpoint():
    await asyncio.sleep(3)  # 대기 중 다른 작업 처리 가능
    return {"message": "done"}
```

## 7. Path Parameter

URL 경로에 포함되는 값

```text
/users/10
```

`10`은 특정 사용자를 식별하는 값

```python
@app.get("/users/{user_id}")
def get_user(user_id: int):
    return {"user_id": user_id}
```

요청

```http
GET /users/10
```

FastAPI가 문자열 `"10"`을 정수 `10`으로 변환한 뒤 함수 실행

```python
get_user(user_id=10)
```

`user_id`가 `int`로 지정되어 있으므로 다음 요청은 검증 실패

```http
GET /users/abc
```

FastAPI가 일반적으로 `422 Unprocessable Content` 응답 반환

## 8. Query Parameter

URL의 `?` 뒤에 전달되는 값

```text
/products?category=book&page=2
```

주로 검색, 필터, 정렬, 페이지네이션에 사용

```python
@app.get("/products")
def get_products(
    category: str | None = None,
    page: int = 1,
):
    return {
        "category": category,
        "page": page,
    }
```

요청

```http
GET /products?category=book&page=2
```

함수 실행

```python
get_products(category="book", page=2)
```

`page`가 전달되지 않으면 기본값 `1` 사용

```python
page: int = 1
```

`category`는 전달하지 않아도 되는 선택값

```python
category: str | None = None
```

## 9. Request Body

생성하거나 수정할 실제 데이터는 Request Body로 전달

FastAPI에서는 Pydantic 모델을 사용해 요청 데이터 구조 정의

```python
from pydantic import BaseModel


class ItemCreate(BaseModel):
    name: str
    price: int
```

```python
@app.post("/items")
def create_item(item: ItemCreate):
    return item
```

요청

```http
POST /items
Content-Type: application/json
```

```json
{
  "name": "Keyboard",
  "price": 50000
}
```

FastAPI의 처리

```text
JSON 요청 수신
→ ItemCreate 객체로 변환
→ 필수 필드와 타입 검증
→ create_item() 실행
```

필수 필드가 없거나 타입이 맞지 않으면 검증 오류 반환

## 10. 입력 방식 비교

| 구분              | 용도            | 예시                   |
| --------------- | ------------- | -------------------- |
| Path Parameter  | 특정 자원 식별      | `/users/10`          |
| Query Parameter | 검색과 필터 조건     | `/users?active=true` |
| Request Body    | 생성하거나 수정할 데이터 | `{"name": "Kim"}`    |

구분 기준

```text
어떤 자원인가
→ Path Parameter

어떤 조건으로 조회하는가
→ Query Parameter

어떤 데이터로 생성하거나 수정하는가
→ Request Body
```

## 11. 실습 코드

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()


# 상품 생성 요청의 JSON 구조
class ItemCreate(BaseModel):
    name: str
    price: int


# 루트 경로 조회
@app.get("/")
def read_root():
    return {"message": "Hello World"}


# 상품 목록 조회: Query Parameter 사용
@app.get("/items")
def read_items(
    keyword: str | None = None,
    page: int = 1,
):
    return {
        "keyword": keyword,
        "page": page,
    }


# 특정 상품 조회: Path Parameter 사용
@app.get("/items/{item_id}")
def read_item(item_id: int):
    return {"item_id": item_id}


# 상품 생성: Request Body 사용
@app.post("/items")
def create_item(item: ItemCreate):
    return {
        "id": 1,  # 데이터베이스가 없으므로 임시 고정값
        "name": item.name,
        "price": item.price,
    }
```

### API 구성

| 기능       | 요청                     | 입력 방식           |
| -------- | ---------------------- | --------------- |
| 루트 경로 조회 | `GET /`                | 없음              |
| 상품 목록 조회 | `GET /items`           | Query Parameter |
| 특정 상품 조회 | `GET /items/{item_id}` | Path Parameter  |
| 상품 생성    | `POST /items`          | Request Body    |

### 상품 목록 조회

요청

```http
GET /items?keyword=keyboard&page=2
```

응답

```json
{
  "keyword": "keyboard",
  "page": 2
}
```

Query Parameter를 생략하면 기본값 사용

```http
GET /items
```

```json
{
  "keyword": null,
  "page": 1
}
```

### 특정 상품 조회

요청

```http
GET /items/10
```

응답

```json
{
  "item_id": 10
}
```

### 상품 생성

요청

```http
POST /items
Content-Type: application/json
```

```json
{
  "name": "Keyboard",
  "price": 50000
}
```

응답

```json
{
  "id": 1,
  "name": "Keyboard",
  "price": 50000
}
```

현재 코드는 데이터베이스를 사용하지 않으므로 모든 생성 요청에 고정된 `id: 1` 반환

실제 서비스에서는 데이터베이스에 저장한 뒤 생성된 ID와 상품 정보 반환

```bash
uvicorn main:app --reload
```
